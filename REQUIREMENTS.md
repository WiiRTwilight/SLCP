# SLCP 需求文档

## 1. 项目概述

| 属性         | 说明                                                         |
| ------------ | ------------------------------------------------------------ |
| **模组名称** | SLCP (Some Long Config Puller)                                |
| **定位**     | 配置驱动的文件下载器，按预设配置在游戏启动时自动下载指定文件   |
| **目标用户** | Minecraft 整合包作者、需要自动更新外部资源的服务器管理员       |
| **运行环境** | Minecraft Java Edition 1.21.11                                |
| **模组加载器** | Fabric Loader >= 0.19.3                                     |
| **Java 版本** | 21                                                            |
| **开源协议** | Apache2.0                                                           |

---

## 2. 功能需求

### FR-1 配置文件管理

| 编号    | 功能点         | 描述                                                         | 优先级 |
| ------- | -------------- | ------------------------------------------------------------ | ------ |
| FR-1.1  | 默认配置释放   | 首次启动时从 JAR 内部将 `slcp_config.json` 释放至 `{游戏配置目录}/slcp/config.json` | P0     |
| FR-1.2  | 配置路径       | 配置文件固定存放在游戏配置目录下的 `slcp/config.json`，不与其它模组冲突 | P0     |
| FR-1.3  | JSON 解析      | 支持 JSON 数组格式，列表内每个对象以 `<名称>:{url, output}` 结构描述一个下载任务 | P0     |
| FR-1.4  | 配置回写       | 提供 `save()` 方法将内存中的配置序列化回 JSON 文件           | P1     |
| FR-1.5  | 日志输出       | 启动时打印已加载的条目数量及每个条目的 `[名称] url -> output` 细节 | P1     |

**配置字段说明：**

| 字段   | 类型   | 说明                           | 必填 |
| ------ | ------ | ------------------------------ | ---- |
| name   | String | 下载任务的标识名称             | 是   |
| url    | String | 下载文件的远程 URL              | 是   |
| output | String | 在游戏目录下的相对输出路径      | 是   |

---

### FR-2 文件下载

| 编号    | 功能点         | 描述                                                         | 优先级 |
| ------- | -------------- | ------------------------------------------------------------ | ------ |
| FR-2.1  | 启动自动下载   | 游戏初始化时（`onInitialize`）自动读取配置并下载所有条目的文件 | P0     |
| FR-2.2  | 手动重下载     | 提供两种触发入口：Mod Menu 界面按钮 + `/slcp redownload` 客户端命令 | P0     |
| FR-2.3  | 文件名提取     | 从 URL 的 path 部分自动提取文件名；若 URL 以 `/` 结尾或无法提取，fallback 为 `download` | P0     |
| FR-2.4  | 输出路径校验   | 下载前校验 `{游戏目录}/{output}/{文件名}` 规范化后仍在游戏目录内，拒绝路径穿越攻击 | P0     |
| FR-2.5  | 目录自动创建   | 下载前自动创建目标路径的父级目录（`Files.createDirectories`） | P1     |
| FR-2.6  | 重定向跟随     | 跟随 HTTP 301/302/303 重定向，最多 5 跳，每次跳转重建连接     | P1     |
| FR-2.7  | User-Agent    | 请求伪装：`Mozilla/5.0 (X11; Linux x86_64; rv:136.0) Gecko/20100101 Firefox/136.0` | P1     |
| FR-2.8  | 超时控制       | 连接超时 30 秒、读取超时 300 秒，兼容大文件下载场景           | P1     |
| FR-2.9  | 容错处理       | 单条目下载失败仅记录错误日志，不影响其余条目的下载             | P1     |
| FR-2.10 | 下载日志       | 每条下载记录起始日志（名称、URL、目标路径）和完成日志（字节数） | P2     |
| FR-2.11 | 流式写入       | 8KB 缓冲区流式写入，避免大文件占满内存                        | P1     |

### FR-3 用户交互

| 编号    | 功能点         | 描述                                                         | 优先级 |
| ------- | -------------- | ------------------------------------------------------------ | ------ |
| FR-3.1  | Mod Menu 集成  | 实现 `ModMenuApi` 接口，在 Mod Menu 中为 SLCP 显示配置图标和入口 | P0     |
| FR-3.2  | 配置屏幕       | 提供两个按钮的配置屏幕：<br>- "Redownload SLCP Files" — 触发重下载<br>- "Done" — 返回上级屏幕 | P0     |
| FR-3.3  | 客户端命令     | `/slcp redownload` 客户端命令，执行与按钮相同的重下载逻辑   | P0     |
| FR-3.4  | 完成反馈       | 下载全部完成后，在玩家聊天栏显示 "SLCP: Redownload complete" 提示 | P1     |
| FR-3.5  | 失败反馈       | 配置未加载时，在聊天栏显示 "SLCP: Config not loaded, cannot redownload" | P1     |
| FR-3.6  | 线程命名       | 下载线程命名为 `SLCP-Redownload`，便于调试和监控              | P2     |

### FR-4 线程模型

| 编号    | 功能点         | 描述                                                         | 优先级 |
| ------- | -------------- | ------------------------------------------------------------ | ------ |
| FR-4.1  | 启动阶段       | 初始化阶段下载在游戏主线程执行（`onInitialize` 阻塞）        | P0     |
| FR-4.2  | 手动触发       | 按钮/命令触发的下载在新线程执行，不阻塞渲染线程，完成后通过 `Minecraft.getInstance().execute()` 切回主线程发送消息 | P0     |

## 4. 配置文件规范

### 4.1 文件位置

```
{游戏配置目录}/slcp/config.json
```

（通过 `Platform.getConfigFolder()` 获取游戏配置目录路径）

### 4.2 默认配置模板

模组 JAR 内 `slcp_config.json` 作为默认模板：

```json
[
  {
    "LICENSE": {
      "url": "https://www.apache.org/licenses/LICENSE-2.0.txt",
      "output": "./"
    }
  }
]
```

### 4.3 格式定义

```json
[
  {
    "<条目名称>": {
      "url": "<文件下载地址>",
      "output": "<游戏目录下的相对路径>"
    }
  }
]
```

- 外层为 JSON 数组，支持多个下载条目
- 每个条目是一个 JSON 对象，键为条目名称，值为包含 `url` 和 `output` 的对象
- `url`：文件的完整下载地址（String）
- `output`：游戏目录下的相对输出路径（String），如 `"./"`、`"mods/"`、`"config/"`

## 5. 界面规范

### 5.1 配置屏幕

| 属性     | 值                                                |
| -------- | ------------------------------------------------- |
| 标题     | "SLCP Configuration"（`screen.slcp.config`）       |
| 布局     | 垂直居中排列                                       |
| 按钮尺寸 | 200 × 20 像素                                      |
| 背景     | 继承 Minecraft 原版 Screen 默认背景                 |

### 5.2 按钮

| 按钮           | 位置 (x, y)                      | 行为                             | 翻译键                     |
| -------------- | -------------------------------- | -------------------------------- | -------------------------- |
| Redownload     | (width/2 - 100, height/2 - 10)   | 触发重下载所有配置文件            | `button.slcp.redownload`   |
| Done           | (width/2 - 100, height/2 + 20)   | 返回上级界面（Mod Menu 列表）     | `gui.done`                 |

### 5.3 命令

```
/slcp redownload
```

- 客户端命令（无需权限）
- 执行效果与 Redownload 按钮完全一致

### 5.4 翻译键

| 键                               | 值 (英文)                                    |
| -------------------------------- | -------------------------------------------- |
| `screen.slcp.config`             | SLCP Configuration                           |
| `button.slcp.redownload`         | Redownload SLCP Files                        |
| `text.slcp.redownload_done`      | SLCP: Redownload complete                    |
| `text.slcp.redownload_failed`    | SLCP: Config not loaded, cannot redownload   |
